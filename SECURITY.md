# Security Policy

## Reporting a Vulnerability

If you discover a security issue in Nyxera Eye, report it privately and do not publish exploit details before triage.

Use the responsible disclosure process documented here:
- `docs/SECURITY_DISCLOSURE.md`

When reporting, include:
- affected component/file path
- reproduction steps
- expected vs actual behavior
- impact assessment
- proof-of-concept details (if available)

## Scope

This policy applies to:
- source code in this repository
- local deployment and runtime behavior
- API/auth/security controls

## Supported Status

Project status is currently R&D/early development.

Security fixes are prioritized for:
- `dev` branch active development baseline
- release branches/tags when present

## Response Expectations

Best-effort process:
1. Acknowledge report
2. Reproduce and triage
3. Determine severity and fix plan
4. Coordinate disclosure timeline

## Safe Harbor

Authorized, good-faith research aligned with project compliance and ethics policies is supported.

Out-of-scope or prohibited activity includes:
- unauthorized access attempts
- brute-force credential attacks
- disruptive testing outside approved scope

See:
- `docs/COMPLIANCE.md`
- `docs/ETHICS.md`
